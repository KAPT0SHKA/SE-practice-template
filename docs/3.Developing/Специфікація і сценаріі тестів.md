## **1. ВХІДНІ СПЕЦИФІКАЦІЇ**

Розділ «Вхідні специфікації» регламентує **входи**, необхідні для проходження всіх тестів для даного рішення. У числі таких входів можуть бути дані, введені з клавіатури, натиснення кнопок миші у відповідних місцях сторінки, датчики, комп'ютерне та інше обладнання, різні дії користувачів. 

Опис специфікацій поряд з найменуванням входу містить його *характеристики*, такі як значення (що введено), структури даних (якого типу), інтерфейси (яким чином), протоколи, бази даних, файли тощо.

| Вхідний елемент        | Характеристики                                         |
|-----------------------|-------------------------------------------------------|
| Дані з клавіатури      | Значення: текстові дані, числові значення              |
|                       | Структури даних: рядки, числа                          |
|                       | Інтерфейс: введення тексту у текстові поля, натискання клавіш |
| Натискання кнопок миші| Значення: координати миші (x, y), клік миші (лівий, правий)|
|                       | Структури даних: пари координат (x, y), тип кліку      |
|                       | Інтерфейс: натискання та рух миші по екрану, клік миші |
| Комп'ютерне обладнання| Значення: стан обладнання (включено/вимкнено), показники роботи |
|                       | Структури даних: логічні значення, числа              |
|                       | Інтерфейс: зчитування даних через відповідні драйвери, протоколи |
| Інші дії користувачів | Значення: дії користувачів (натискання клавіш, перегляд вмісту, тощо) |
|                       | Структури даних: рядки, логічні значення               |
|                       | Інтерфейс: сприйняття дій користувача через веб-інтерфейс, клавіатуру, мишу тощо |

## **2. ВИХІДНІ СПЕЦИФІКАЦІЇ**

Розділ «Вихідні специфікації» регламентує **виходи**, очікувані як результат проходження тестів для вашого рішення. Виходи можуть містити дані, інформацію про час виконання операцій, повідомлення, логи, дампи пам'яті, файли, знімки екрану тощо. 

| Вихідний елемент      | Опис                                                           |
|-----------------------|----------------------------------------------------------------|
| Графічний вихід       | Зображення ігрового середовища, персонажів, об'єктів, ефектів |
| Логі дій гравця       | Інформація про дії гравця, які відображаються в ігровому журналі |
| Результати гри        | Очки, рівень, досягнення, пройдені етапи гри                  |
| Повідомлення про помилки | Інформація про помилки або проблеми під час гри, яка відображається користувачеві |
| Логи                 | Інформація про події, помилки, важливі відомості під час виконання гри |
| Файли результатів   | Файли збереження результатів гри або стану гравця              |
| Аудіо вихід          | Звукові ефекти, музика, голосові повідомлення під час гри      |

## **5. ВЗАЄМОЗАЛЕЖНОСТІ**

Функціональне тестування може впливати на тестування інтерфейсу користувача (UI) та тестування користувальницького досвіду (UX), оскільки правильна робота функцій системи впливає на спосіб, яким користувачі сприймають і взаємодіють з інтерфейсом.

Тестування продуктивності може впливати на навантажувальне тестування, оскільки тестування ефективності системи за різних навантажень може виявити проблеми з її продуктивністю та стійкістю.

Тестування безпеки може впливати на надійність тестування, оскільки вразливості системи можуть призвести до збоїв або втрати даних, що вплине на її надійність.

Тестування сумісності може впливати на тестування продуктивності, оскільки різні пристрої та платформи можуть мати різні характеристики, що впливає на продуктивність програми або системи.

## **6. ТЕСТИ**
Цей розділ присвячений опису конкретних тестів. В кожному з його підпунктів описано тестування певної частини розробленої функціональності. При цьому формулюються такі характеристики як *очікувана поведінка* (що саме перевіряємо), *очікувана швидкодія* (скільки часу має тривати операція), очікувана достовірність (у відсотках), зауваження, які виникли в процесі тестування.

### **6.1. Тест 1 - Перевірка валідності введених даних**

**Назва властивості A: Перевірка на наявність обов'язкових полів**

ОЧІКУВАНА ПОВЕДІНКА: Система повинна виявити відсутність обов'язкових полів і відобразити відповідне повідомлення про помилку.

ОЧІКУВАНА ШВИДКОДІЯ: Перевірка має здійснюватися миттєво, максимальний час очікування - 1 секунда.

ОЧІКУВАНА ДОСТОВІРНІСТЬ: 100% - система повинна правильно визначати наявність обов'язкових полів.

ЗАУВАЖЕННЯ: Важливо перевірити, що всі обов'язкові поля на формі визначені як обов'язкові як на клієнтському, так і на серверному рівнях.

**Назва властивості B: Перевірка формату введених даних**

ОЧІКУВАНА ПОВЕДІНКА: Система повинна перевірити правильність формату введених даних (наприклад, електронної пошти, номера телефону) і відобразити повідомлення про помилку при неправильному форматі.

ОЧІКУВАНА ШВИДКОДІЯ: Перевірка має здійснюватися миттєво, максимальний час очікування - 1 секунда.

ОЧІКУВАНА ДОСТОВІРНІСТЬ: 95% - система повинна правильно визначати коректність формату введених даних.

ЗАУВАЖЕННЯ: Важливо врахувати різноманітні варіанти формату даних і перевірити їх на відповідність вимогам. Наприклад, при перевірці електронної пошти врахувати використання спеціальних символів, символу "@" та наявність домену.

### **6.2. Тест 2 - Перевірка функціоналу входу в систему**

**Назва властивості A: Перевірка правильності логіну та паролю**

ОЧІКУВАНА ПОВЕДІНКА: Система повинна перевірити відповідність введених логіну та паролю збереженим в базі даних і, в разі відповідності, надати доступ користувачеві.

ОЧІКУВАНА ШВИДКОДІЯ: Перевірка має здійснюватися швидко, максимальний час очікування - 2 секунди.

ОЧІКУВАНА ДОСТОВІРНІСТЬ: 99% - система повинна правильно визначати відповідність введених даних.

ЗАУВАЖЕННЯ: Важливо забезпечити безпеку процесу входу в систему шляхом використання шифрування паролів та заходів захисту від перебору паролів.

**Назва властивості B: Перевірка обмежень на спроби входу**

ОЧІКУВАНА ПОВЕДІНКА: Система повинна обмежувати кількість спроб входу в систему з неправильними даними та блокувати доступ на певний час після досягнення певного обмеження.

ОЧІКУВАНА ШВИДКОДІЯ: Перевірка має здійснюватися миттєво, максимальний час очікування - 1 секунда.

ОЧІКУВАНА ДОСТОВІРНІСТЬ: 100% - система повинна правильно виконувати обмеження на спроби входу.

ЗАУВАЖЕННЯ: Важливо налаштувати параметри обмежень входу в систему з урахуванням безпеки та зручності користувачів. Наприклад, блокування може бути тимчасовим і з різними термінами розблокування для різних кількостей невдалого входу.
